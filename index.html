About me

I am a highly motivated and quality-driven QA Engineer with over 3 years of experience delivering end-to-end software testing solutions across banking, healthcare, and digital industries. I bring strong expertise in both manual and automation testing, with hands-on experience in building scalable UI automation frameworks using Playwright (JavaScript) and Katalon Studio.

With a strong analytical mindset and automation-first approach, I am passionate about continuously improving testing processes, enhancing automation coverage, and contributing to engineering teams that value quality, efficiency, and innovation. Recruiter FAQ

Quick answers to common questions about my QA & automation experience, tools, and working style.

1. What is your main QA expertise?

I focus on end-to-end software testing across web and mobile, including manual testing, UI automation, API testing, performance testing, and database validation. I am experienced in SIT, UAT, regression, smoke testing, and BVT.

2. Which automation tools do you use?

For modern UI automation I use Playwright (JavaScript). For enterprise automation workflows I use Katalon Studio. I apply reusable design patterns such as Page Object Model to keep tests scalable and maintainable.

3. How do you ensure quality in SIT/UAT and releases?

I collaborate closely with developers and business stakeholders to validate requirements, prepare test plans, execute regression suites, and communicate risk-based quality recommendations before release. I track execution and defects using tools like JIRA and qTest.

4. Do you have performance testing experience?

Yes. I use JMeter to run load and performance testing, simulate high concurrency, and analyze response time, throughput, and error rates to support stability and capacity decisions.

5. How do you validate backend data?

I perform database validation to ensure data integrity and accuracy using SQL queries (SQL Server and AS400). This helps confirm that UI/API results match backend records and business rules.
